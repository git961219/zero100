# Zero100 UX/UI 기획안
> GPS 기반 제로백(0-200km/h) 측정 안드로이드 앱
> 작성: 기획팀장 | 2026-04-18 | v1.0

---

## 1. 전체 사용 흐름도 (User Flow)

```
[앱 실행]
    |
    v
[메인 화면] --- GPS 모듈 미연결 ---> [연결 안내 다이얼로그]
    |
    | USB OTG 감지 (자동)
    v
[GPS 연결중] --- 위성 탐색 ---+
    |                          |
    | 안정화 조건 충족          | 미충족 (타임아웃 60초)
    | (25Hz + 위성>=10 +       |
    |  HDOP<2.0)               v
    v                    [경고: 정밀도 부족]
[측정 준비 완료]              (측정 가능하나 경고 표시)
    |
    | "측정 시작" 탭
    v
[정차 대기] --- 속도 > 2km/h ---> [정차 안내 재표시]
    |
    | 속도 = 0 (3초 유지)
    v
[출발 대기] --- "지금 출발하세요"
    |
    | 속도 > 0 감지 (자동 트리거)
    v
[측정 중] --- 목표 속도 도달 또는 수동 종료
    |
    v
[결과 화면] --- 저장/공유/재측정
    |
    v
[기록 목록] (메인에서도 접근 가능)
```

## 3. 공통 컴포넌트

### 3-1. GPS 상태 바 (Global Top Bar)

```
+----------------------------------------------------------+
| [위성 아이콘] 14/14  |  25Hz  |  HDOP 0.8  |  Fix: 3D   |
+----------------------------------------------------------+
```

- 높이: 32dp
- 배경: 상태별 색상
  - 빨강: 미연결 / 위성 < 6 / HDOP > 3.0

- 탭하면 GPS 상세 정보 바텀시트 확장

## 4. 화면별 상세 레이아웃

### 4-1. 메인 화면 (Dashboard)

**목적**: 현재 상태 한눈에 파악 + 빠른 측정 진입

```
+----------------------------------------------------------+
| [GPS 상태 바 - 초록]                            [설정]    |
+----------------------------------------------------------+
|                                                          |
|                      +-----------+                       |
|                      |           |                       |
|                      |   0.0     |   <-- 현재 속도       |
|                      |  km/h     |       (초대형 폰트    |
|                      |           |        120sp)         |
|                      +-----------+                       |
|                                                          |
|  +----------------------------------------------------+  |
|  |  최고 기록                                          |  |
|  |  0-100: 4.32s    0-200: 12.87s    최고속도: 243km/h |  |
|  +----------------------------------------------------+  |
|                                                          |
|  +----------------------------------------------------+  |
|  |                                                    |  |
|  |          [    측정 시작    ]                        |  |
|  |          (대형 원형 버튼, 강조색)                   |  |
|  |                                                    |  |
|  +----------------------------------------------------+  |
|                                                          |
|  GPS 미연결 시: 버튼 -> "GPS 연결" (비활성 느낌)         |
|  안정화 미달 시: 버튼 -> "GPS 안정화 중..." (펄스 애니)  |
|  준비 완료 시: 버튼 -> "측정 시작" (강조색 + 글로우)     |
|                                                          |
+----------------------------------------------------------+
|   [계기판]        [측정]          [기록]                  |
+----------------------------------------------------------+
```

**설계 포인트**:
- 속도 숫자는 화면 중앙, 최대한 크게 (운전 중 시인성)
- 측정 시작 버튼은 엄지 도달 영역 (하단 40% 이내)
- GPS 상태에 따라 버튼 상태 자동 전환 (수동 연결 불필요)
- 최고 기록은 동기부여 요소로 항상 노출

### 4-2. 측정 화면 (Measure)

**목적**: 운전 중 최소 시선 이동으로 핵심 정보 파악

#### Phase 1: 정차 대기

```

+----------------------------------------------------------+

|          +----------------------------------+            |
|                                                          |
|          속도가 0이 되면 자동으로 다음 단계              |
|                                                          |
|                 [ 취소 ]                                 |
+----------------------------------------------------------+
```

#### Phase 2: 출발 대기

```
+----------------------------------------------------------+
| [GPS 상태 바]                                             |
+----------------------------------------------------------+
|                                                          |
|          +----------------------------------+            |
|          |     [ >> 화살표 애니메이션 ]      |            |

|          |                                  |            |
|          |   목표: 0 -> 100 km/h           |            |
|          +----------------------------------+            |

|                                                          |
|                 [ 취소 ]                                 |
+----------------------------------------------------------+
```

#### Phase 3: 측정 중 (핵심 화면)

```
+----------------------------------------------------------+
| [GPS 상태 바]                              [측정중] 빨강  |
+----------------------------------------------------------+
|                                                          |
|                    4.327                                  |
|                      s            <-- 경과 시간          |
|                                       (초대형, 100sp)    |
|                                                          |
|                   87.3                                   |
|                  km/h             <-- 현재 속도          |
|                                       (대형, 64sp)       |
|                                                          |
|  +----------------------------------------------------+  |
|  |  구간 랩타임                                        |  |
|  |                                                    |  |
|  |  0-60   [  2.14s  ] ----+                          |  |
|  |  0-100  [  ----   ]     |  프로그레스 바           |  |
|  |  0-150  [  ----   ]     |  (현재 진행률 표시)      |  |
|  |  0-200  [  ----   ]  ---+                          |  |
|  +----------------------------------------------------+  |
|                                                          |
|  +----------------------------------------------------+  |
|  |  [실시간 속도 그래프 - 미니]                        |  |
|  |  ^                    .../                          |  |
|  |  |              .....                               |  |
|  |  |         ....                                     |  |
|  |  |    ...                                           |  |
|  |  +--------------------------->                      |  |
|  +----------------------------------------------------+  |
|                                                          |
|                [ 수동 종료 ]                              |
+----------------------------------------------------------+
```

**설계 포인트**:
- 경과 시간이 가장 큰 폰트 (드라이버가 가장 궁금한 정보)
- 현재 속도는 두 번째 크기
- 구간 랩은 도달 시 숫자가 "팝" 애니메이션으로 등장
- 미도달 구간은 "----"로 표시
- 프로그레스 바: 현재 속도/목표 속도 비율로 시각적 진행률
- 그래프는 최소 높이 (있으면 좋지만 운전 중 안 봄)
- 수동 종료 버튼은 작게, 실수 방지

## 5. GPS 안정화 로직 & UX

### 5-1. 안정화 판정 조건

| 조건 | 임계값 | 우선도 |
|------|--------|--------|
| GPS Fix | 3D Fix | 필수 |
| 갱신율 | 25Hz 연속 3초 | 필수 |
| 위성 수 | >= 10개 | 필수 |
| HDOP | < 2.0 | 필수 |
| 속도 안정성 | 정지 시 +/- 0.5km/h 이내 (5초) | 권장 |

### 5-2. 안정화 UX 시퀀스

```
[Phase 0] GPS 미연결
  - 상태바: 빨강
  - 메인 버튼: "GPS를 연결하세요" (회색, 비활성)
  - USB OTG 연결 시 자동 감지 -> Phase 1

[Phase 1] 위성 탐색 중
  - 상태바: 빨강 -> 주황 (위성 수 증가에 따라)
  - 메인 버튼: "위성 탐색 중... (6/10)" (펄스 애니메이션)
  - 프로그레스: 원형 게이지로 4개 조건 달성률 시각화

[Phase 2] 안정화 진행 중
  - 상태바: 주황
  - 메인 버튼: "안정화 중... (3/4 조건 충족)"
  - 미충족 조건 하이라이트 표시

[Phase 3] 측정 가능
  - 상태바: 초록 (색상 전환 애니메이션)
  - 메인 버튼: "측정 시작" (강조색 + 글로우 이펙트)
  - 햅틱 피드백 (진동 1회)
  - 조건 충족 후 바로 전환 (딜레이 없음)
```

### 5-3. 안정화 프로그레스 표시 (상세)

```
        +------ 안정화 상태 ------+
        |                         |
        |   [O] Fix: 3D      OK  |
        |   [O] 위성: 14/10  OK  |
        |   [X] HDOP: 2.3   >2.0 |
        |   [O] Hz: 25/25    OK  |
        |                         |
        |   3/4 조건 충족         |
        +-------------------------+
```

## 6. 비주얼 디자인 가이드라인

### 6-1. 색상 팔레트 (다크 모드 기본)

| 용도 | 색상 | HEX |
|------|------|-----|
| 배경 (Primary) | 거의 검정 | #0D0D0D |
| 배경 (Surface) | 진한 회색 | #1A1A1A |
| 카드 배경 | 회색 | #242424 |
| 주요 텍스트 | 밝은 흰 | #F0F0F0 |
| 보조 텍스트 | 중간 회색 | #888888 |
| 강조색 (Accent) | 레이싱 레드 | #FF3B30 |
| 보조 강조색 | 네온 그린 | #30D158 |
| 경고 | 앰버 | #FF9500 |
| 비활성 | 어두운 회색 | #3A3A3A |
| 그래프 라인 | 밝은 시안 | #00D4FF |

### 6-2. 타이포그래피

| 용도 | 서체 | 크기 | 가중치 |
|------|------|------|--------|
| 메인 속도 표시 | Mono (예: JetBrains Mono) | 120sp | Bold |
| 타이머 | Mono | 100sp | Bold |
| 구간 랩타임 | Mono | 28sp | Medium |
| 일반 텍스트 | Pretendard | 16sp | Regular |
| 보조 텍스트 | Pretendard | 14sp | Regular |
| 라벨 | Pretendard | 12sp | Medium |

- 숫자는 반드시 모노스페이스 (자릿수 변해도 레이아웃 흔들림 없음)
- 속도/시간은 고정폭 + 우측 정렬

### 6-3. 디자인 원칙

1. **미니멀 레이싱**: 불필요한 장식 배제, 정보 밀도 높게
2. **계층 명확**: 가장 중요한 숫자 1개가 즉시 눈에 들어와야 함
3. **고대비**: WCAG AAA 수준, 직사광선에서도 가독
4. **AMOLED 최적화**: 순수 검정 배경 옵션 (#000000)으로 배터리 절약
5. **애니메이션 최소화**: 측정 중에는 불필요한 애니메이션 배제 (성능/배터리)
6. **가로 모드 미지원**: 세로 고정 (대시보드 거치 시 세로가 일반적)

## 7. 1-Foot Rollout 보정 UX

### 7-1. 설명 텍스트 (설정 내)

```
1-Foot Rollout 보정

드래그 레이싱에서 차량이 스테이징 라인의 타이밍 빔을
통과하기까지의 1피트(약 30cm) 이동 시간을 보정합니다.

- ON: 차량 잡지/드래그 스트립 기준과 동일 (약 0.1~0.3초 빠름)
- OFF: 순수 0km/h 기준 측정 (GPS 원본)

* 자동차 매체 비교 시 ON 권장
```

### 7-2. 결과 화면 표시

```
0-100 km/h: 4.327s
  (rollout 보정: -0.08s, 보정 전: 4.407s)
```

- 보정 ON일 때 보정량을 투명하게 표시
- 사용자가 두 값을 모두 확인 가능

| 항목 | 타임슬립 카드 (보수적) | 영상 오버레이 (과감) |
|------|----------------------|---------------------|

| 리스크 | 낮음 | 카메라+GPS 동기화 이슈 |
| **판정** | MVP 이후 즉시 | v2.0 검토 |

### 9-3. "지금 안 하면 나중에 비싸지는 것"

1. **데이터 구조 설계**: 측정 결과 저장 스키마를 처음부터 확장 가능하게 설계해야 함. 나중에 브레이킹, 커스텀 구간, G-force 등 추가할 때 마이그레이션 비용 급증.
   - 권장: 측정 raw data (25Hz 전체 GPS 포인트)를 별도 저장 -> 나중에 재분석 가능
2. **NMEA 파싱 라이브러리 분리**: GPS 모듈 교체 가능성 대비. u-blox M9 전용 코드가 아닌 추상화 레이어 필요.
3. **그래프 라이브러리 선정**: v1.0에서 간단한 그래프라도 넣어야 v1.5에서 비교 오버레이를 자연스럽게 추가 가능. 나중에 넣으면 라이브러리 통합 비용 발생.
